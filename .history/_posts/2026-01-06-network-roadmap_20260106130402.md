## 0. 네트워크 공부를 하자

진행중인 사이드 프로젝트의 서버 세팅을 혼자서 진행하면서, 재미와 부족함을 함께 느꼇다.  
막연한 두려움을 가졌던 cmd 환경에서 명령어로 작업하는 리눅스 서버 세팅은 생각보다 재밌었지만,  
반대로 서버 관련 세팅을 하면서 개념만 간단하게 알고 있던 네트워크 용어들이 자꾸 걸렸다.  
앞으로 진행할 리소스 관리 및 패킷 분석을 위해서는 네트워크 개념을 좀 더 확실히 정립할 필요를 느꼇다.

## 1. 네트워크 로드맵 

내가 공부할 네트워크 로드맵을 소개하겠다.  
단계별로 공부하는 것이 목적이다. 

📍 1. 기초 다지기 (모델과 구조)
네트워크가 데이터를 주고받는 이론적/실제적 틀을 이해.

- OSI 7계층 & TCP/IP 4계층: 각 계층의 역할과 차이점 이해

- 캡슐화 & 역캡슐화: 데이터 전송 시 헤더가 붙고 떨어지는 과정

- 패킷(Packet): 네트워크 데이터 전송의 기본 단위 이해

- 주소 체계: MAC 주소(물리) vs IP 주소(논리)의 역할

📍 2. 전송과 주소 지정 (L3 & L4)
가장 중요한 '어떻게 연결하고 보내는가'에 대한 핵심 로직.

- IP (Internet Protocol): IPv4/IPv6, 서브넷 마스크(Subnet Mask), 공인/사설 IP, NAT

- TCP (연결 지향): 3-Way & 4-Way Handshake, 흐름 제어, 혼잡 제어

- UDP (비연결 지향): TCP와의 차이점 및 실시간 스트리밍 적용 사례

- 포트(Port) 번호: 서비스별 고유 번호 (HTTP: 80, HTTPS: 443 등)

📍 3. 웹과 애플리케이션 (L7)
개발자가 실무에서 가장 많이 다루는 웹 통신 영역.

- HTTP 프로토콜: HTTP/1.1 vs HTTP/2 vs HTTP/3(QUIC) 차이점

- DNS (Domain Name System): 도메인 이름이 IP 주소로 변환되는 과정

- HTTPS & 보안: SSL/TLS 핸드셰이크, 대칭키/공개키 암호화 원리

- API 통신: REST, gRPC, WebSocket의 특징과 선택 기준

📍 4. 시스템 아키텍처 및 인프라
서버 성능과 가용성을 높이기 위한 네트워크 설계 지식.

- 로드 밸런서 (L4/L7): 트래픽 분산 처리의 원리와 알고리즘

- 리버스 프록시 (Reverse Proxy): Nginx 등을 활용한 보안 및 캐싱

- CDN (Content Delivery Network): 전 세계 사용자에게 빠른 콘텐츠 전달 원리

- VPC & 보안 그룹: 클라우드(AWS 등) 환경에서의 가상 네트워크 설계
